# PRD-孕期健康伙伴 V1.2 技术方案（V1 本地优先）

日期：2026-02-23

## 1. 技术栈
- 前端：React Native
- 本地存储：SQLite（建议）或 AsyncStorage（仅轻量）
- 通知：iOS/Android 本地通知
- AI：DeepSeek API（仅解析意图与结构化数据）

## 2. 本地存储选型

### 2.1 SQLite（推荐）
优点：
- 结构化数据强、可查询、可扩展
- 适合提醒/检查记录/时间线查询

建议方案：
- React Native: `react-native-sqlite-storage` 或 `expo-sqlite`（若使用 Expo）
- 通过 DAO 封装 CRUD

### 2.2 AsyncStorage（仅轻量配置）
用途：
- 存用户偏好（例如三餐/睡眠时间）
- 不建议存大量记录

## 3. 通知与提醒策略

### 3.1 本地通知调度
- 每个 ReminderItem 创建通知
- 频率：一次性、每日、每周
- 提醒触发后，若未确认，30 分钟后再发一次“追问通知”

### 3.2 确认与追问逻辑
- 提醒弹窗含“已服用/已完成”按钮
- 若用户未确认，30 分钟后追问一次
- 追问后不再重复

## 4. AI 解析接入

### 4.1 请求结构
- 输入：用户文本 + 用户档案摘要（孕周/三餐/睡眠时间）
- 输出：意图 + 槽位 + 是否需要追问

### 4.2 System Prompt 要求
- 只输出结构化 JSON
- 不输出医学诊断
- 不做治疗建议

### 4.3 结构化结果示例
```json
{
  "intent": "create_medication",
  "slots": {
    "item_name": "钙片",
    "frequency": "daily",
    "time_semantic": "晚饭后",
    "duration_days": 30
  },
  "need_clarify": false
}
```

## 5. 时间语义映射

规则：
- 用户首次引导设置三餐/睡眠时间
- 饭后 = 对应餐点时间 +20 分钟
- 睡前 = 睡觉时间 -30 分钟
- 若用户给出具体时间，优先使用具体时间

## 6. 任务落地顺序（建议）
1. 数据模型 + SQLite schema
2. 首页卡片流 + 输入框
3. AI 解析与意图写入
4. 本地通知调度
5. 今日时间线 / 检查记录页
6. 家属查看只读页

## 7. 风险点与规避
- 参考范围来源：仅使用报告单
- AI 误判：强制澄清机制
- 数据安全：默认本地存储，不上传云端
