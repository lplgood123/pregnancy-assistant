# 孕期助手后端部署（腾讯云轻量服务器，小白步骤）

> 目标：把后端从 Render 迁到腾讯云，减少“启动中/超时”。
> 
> 你现在的 App 已支持多后端 URL 兜底，部署完成后可设置“腾讯云优先，Render 备用”。

## 0. 你需要准备

1. 腾讯云账号（已实名认证）。
2. 一台轻量服务器（推荐 `中国香港` 地域，Ubuntu 22.04）。
3. 你自己的 API Key（Minimax 或 Gemini OpenAI 兼容接口）。
4. 终端可用（你已经在 Mac 上可用）。

## 1. 购买并创建服务器

1. 打开腾讯云控制台 -> 轻量应用服务器。
2. 新建实例：
   - 地域：`中国香港`（测试阶段最省心，不涉及备案）。
   - 镜像：`Ubuntu 22.04`。
   - 套餐：先选最低配测试套餐即可。
3. 创建后，记录：
   - 公网 IP
   - root/ubuntu 登录方式（密码或密钥）

## 2. 开放防火墙端口

在轻量服务器防火墙里放行：
1. `22`（SSH）
2. `80`（HTTP）
3. `443`（HTTPS）

## 3. 连接服务器并安装运行环境

### 3.1 SSH 登录

在你的 Mac 终端执行：

```bash
ssh ubuntu@你的服务器公网IP
```

如果用户名不是 `ubuntu`，按控制台实际用户名替换。

### 3.2 一键安装 Docker

先把本仓库里的脚本上传到服务器，最简单做法：先在服务器 clone 代码，然后执行脚本。

```bash
# 服务器上执行
cd ~
git clone https://github.com/lplgood123/pregnancy-assistant.git
cd pregnancy-assistant/ai_backend/deploy/tencent
./setup_server.sh
```

执行完成后，退出 SSH 并重新登录一次（让 docker 用户组生效）。

## 4. 填写后端环境变量

重新登录服务器后执行：

```bash
cd ~/pregnancy-assistant/ai_backend/deploy/tencent
cp .env.example .env
nano .env
```

必须改的项：

1. `DOMAIN=`
   - 测试可先填：`你的公网IP.sslip.io`
   - 例如 IP 是 `1.2.3.4`，就填 `1.2.3.4.sslip.io`
2. `AI_BACKEND_TOKEN=`
   - 填一个长随机字符串（后面 Xcode 要用同一值）
3. `MINIMAX_API_KEY=`
   - 填你正在使用的供应商 Key
4. 模型相关（二选一）：
   - Minimax：
     - `MINIMAX_BASE_URL=https://api.minimax.io/v1`
     - `MINIMAX_MODEL=MiniMax-M2.5`
   - Gemini（OpenAI 兼容）：
     - `MINIMAX_BASE_URL=https://generativelanguage.googleapis.com/v1beta/openai`
     - `MINIMAX_MODEL=gemini-2.5-flash`

## 5. 启动服务

```bash
cd ~/pregnancy-assistant/ai_backend/deploy/tencent
./deploy_backend.sh
```

如果成功，你会看到容器状态，并且能访问：

- 本机检查：`http://127.0.0.1:8787/healthz`
- 公网检查：`https://你的DOMAIN/healthz`

## 6. 在 Xcode 配置 App 指向新后端

打开项目后，在 Target -> Build Settings 填：

1. `INFOPLIST_KEY_AI_BACKEND_URL`
   - 推荐填成双地址兜底：
   - `https://你的DOMAIN,https://pregnancy-ai-backend.onrender.com`
2. `INFOPLIST_KEY_AI_BACKEND_TOKEN`
   - 填你 `.env` 里的 `AI_BACKEND_TOKEN`
3. `INFOPLIST_KEY_AI_BACKEND_MODEL`
   - 与后端模型一致，例如：`MiniMax-M2.5` 或 `gemini-2.5-flash`

## 7. 真机验证

1. 安装到 iPhone。
2. 首页发送“你好”。
3. 点击“今日安排”，应直接发送并返回。
4. 若失败，先在服务器看日志：

```bash
cd ~/pregnancy-assistant/ai_backend/deploy/tencent
docker compose --env-file .env -f docker-compose.yml logs -f backend
```

## 8. 以后如何更新后端代码

每次更新后端时，在服务器执行：

```bash
cd ~/pregnancy-assistant
git pull
cd ai_backend/deploy/tencent
./deploy_backend.sh
```

这会自动重新构建并重启，不影响你 App 本地数据（每台手机记忆仍独立）。
