# PRD-孕期健康伙伴 V1.2 研发任务清单（可执行）

日期：2026-02-23

## 0. 项目准备
- 确认 RN 版本与是否使用 Expo
- 建立本地开发环境与运行脚本
- 定义代码目录结构（screens/components/services/storage）

## 1. 数据层
1.1 定义数据模型与字段
- UserProfile
- ReminderItem
- CheckRecord
- Conversation
- FamilyShare

1.2 SQLite Schema 设计
- 建表与索引
- 版本号与迁移策略

1.3 数据访问层（DAO）
- 创建、查询、更新、删除
- 封装统一接口

## 2. AI 解析模块
2.1 System Prompt 定义
- 仅输出 JSON
- 不做诊断

2.2 解析服务（Service）
- 调用 DeepSeek API
- 返回意图与槽位
- 处理 need_clarify

2.3 澄清逻辑
- 缺少频率/时间时自动追问
- 多轮输入合并

## 3. 时间语义映射
- 读取用户三餐/睡眠时间
- 语义时间 → 具体时间（饭后 +20 / 睡前 -30）
- 具体时间优先级高于语义时间

## 4. 通知与提醒
4.1 本地通知调度
- 创建提醒
- 取消/更新提醒

4.2 追问机制
- 未确认 30 分钟后发送追问
- 追问后停止

4.3 已完成确认
- 按钮“已服用/已完成”
- 写入记录并停止追问

## 5. 首页 / 对话页
- 卡片流布局
- 常驻输入框
- AI 解析结果确认提示

## 6. 今日时间线页
- 按时间排序
- 完成状态标记
- 一键完成

## 7. 检查记录页
- 列表页（按类型或时间）
- 详情页（本次 vs 上次）
- 有/无参考范围的展示逻辑

## 8. 个人档案页
- 三餐/睡眠时间设置
- 基础信息录入

## 9. 家属查看页（只读）
- 今日提醒摘要
- 检查记录摘要

## 10. 质量与验收
- 用药录入 → 提醒 → 追问 → 完成闭环
- 检查记录录入 → 趋势对比
- 语义时间映射正确

---

## 任务分批建议

### Phase 1（核心闭环）
- 数据层
- AI 解析服务
- 时间语义映射
- 通知提醒 + 追问
- 首页输入 + 今日时间线

### Phase 2（记录与展示）
- 检查记录页
- 个人档案页

### Phase 3（辅助功能）
- 家属查看页
- 空状态与文案优化
