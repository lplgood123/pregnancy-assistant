# PRD-孕期健康伙伴 V1.2 交互流程与AI解析

日期：2026-02-23

## 1. 首屏卡片结构（对话首页）
目标：非传统目录，用户一眼看到“今天要做什么”，并能直接输入。

卡片流顺序（建议）：
1. 今日关怀卡
- 内容：一句问候 + 今日关键事项数量（如“今天还有 3 件事”）。
- 操作：点击进入“今日时间线”。

2. 今日提醒卡（优先级最高）
- 内容：下一个即将到点的提醒（药品/就医/习惯）。
- 操作：按钮“已完成/已服用”；未点击则 30 分钟后追问一次。

3. 今日注意事项卡
- 内容：饮食/休息/运动/禁忌等条目，可勾选完成。
- 操作：勾选完成或长按编辑。

4. 最近检查记录卡
- 内容：最近一次妊娠三项结果摘要 + 趋势箭头。
- 操作：点击进入详情对比。

5. 快速记录卡
- 内容：常用模板入口（“妊娠三项”“产检预约”“用药”）。
- 操作：一键打开对话或表单。

底部常驻输入框：
- 占据视觉中心，支持自然语言录入。
- 输入提示：例如“晚饭后吃钙片”“今天妊娠三项是…”。

## 2. AI 解析规则（核心意图与槽位）

### 2.1 意图列表
- `create_medication`: 新增用药/补剂事项
- `update_medication`: 修改已有用药时间/频率/剂量
- `create_check_record`: 新增检查记录
- `update_check_record`: 修改检查记录
- `create_reminder`: 新增非用药提醒（如产检预约）
- `confirm_intake`: 确认已服用/已完成
- `query_schedule`: 查询当天/本周待办

### 2.2 关键槽位
用药/提醒相关：
- `item_name`（药品/事项名称）
- `dosage`（剂量，可选）
- `frequency`（每天/每周/一次性）
- `time_semantic`（饭后/饭前/睡前/起床后）
- `time_exact`（若用户给出具体时间）
- `duration`（连续几天/几周）

检查记录相关：
- `check_type`（妊娠三项/NT/唐筛等）
- `check_date`
- `values`（指标名-数值-单位）
- `ref_ranges`（报告单参考范围，可选）

### 2.3 语义时间映射规则
- 用户首次引导设置：早餐/午餐/晚餐/睡觉时间。
- 默认偏移：
  - 饭后：+20 分钟
  - 睡前：-30 分钟
- 若用户提供具体时间，优先使用具体时间。

### 2.4 澄清机制
- 语义不清晰时先追问确认：
  - 例：“你是每天都要吃，还是只今天？”
- 若仍不清晰，转为表单录入。

### 2.5 批量指令
- 支持：
  - “早晚各一次，连吃 7 天”
  - “每天三次，饭后服用”

## 3. 关键流程文案（V1）

### 3.1 用药录入（自然语言）
用户：
“晚饭后我要吃钙片，帮我记录一下。”

系统解析：
- 事项：钙片
- 频率：每日
- 时间：晚饭后（映射为晚饭后 +20 分钟）

系统确认：
“已为你创建：每天晚饭后 20 分钟吃钙片。需要改时间吗？”

### 3.2 妊娠三项录入
用户：
“我今天妊娠三项检查是 HCG 12000，孕酮 18，E2 320。”

系统解析并追问（若缺单位/参考范围）：
“已记录。方便补充报告单参考范围吗？没有也可以。”

系统保存后展示：
- 本次结果
- 与上次趋势对比
- 若无参考范围：仅提示趋势

### 3.3 提醒追问
提醒触发：
“该吃钙片啦。”

若 30 分钟未确认：
“刚才的钙片吃了吗？如果没吃，我可以再提醒一次。”

### 3.4 家属查看
家属端或共享视图：
- 只读展示今日提醒 + 检查记录摘要
- 不允许修改与删除

## 4. 风险边界提示
- 不做医学诊断，只做记录与趋势对比。
- 异常提示仅引导咨询医生。
- 参考范围以报告单为准。
