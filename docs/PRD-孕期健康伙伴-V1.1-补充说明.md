# PRD-孕期健康伙伴 V1.1 补充说明

日期：2026-02-23

## 已确认范围与策略
- 目标人群：孕期全覆盖（备孕到产后 42 天）。
- V1 家属查看：需要（只读为主）。
- 语义时间：由用户设置三餐时间与作息时间，语义时间按用户设置映射。
- AI 兜底：先追问澄清，仍不清晰再转手动表单。
- 批量指令：允许（例如“早晚各一次，连吃 7 天”）。
- 数据存储：V1 本地存储。
- 主界面形态：允许底部 Tab，但首页为卡片流 + 常驻输入框，弱化目录感。

## 功能落地细化（需要写回主 PRD）
### 语义时间映射
- 用户首次引导填写三餐时间与睡眠时间。
- 语义时间（饭前/饭后/睡前/起床后）映射为用户时间基准 + 偏移量。
- 偏移量未确认，先标记为待定（见下方“待确认”）。

### 提醒策略
- V1 提醒后若用户未点击“已服用/已完成”，需要追问一次（追问间隔待定）。

### 检查记录与参考范围
- 检查记录字段优先按报告单原样录入。
- 参考范围优先来自报告单；若计划内置参考范围，需要明确权威来源与更新机制。

### AI 解析与安全边界
- “增/记”自动执行；“删/改”需确认。
- 不作医学诊断；异常仅建议咨询医生。

## 待确认
1. 追问间隔时间：30 分钟。
2. 语义时间偏移量默认值：
   - 饭后：20 分钟。
   - 睡前：30 分钟。
3. 参考范围来源策略：仅使用报告单参考范围。

- 将以上补充项合并进主 PRD（建议新版本 V1.2）。
- 明确提醒确认机制与语义时间偏移量，避免开发阶段反复改动。
